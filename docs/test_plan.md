# 法律法规智能问答系统测试计划（Day 1）

## 1. 文档信息

- 项目名称：legal-qa-system
- 负责人：组员 D
- 文档路径：`docs/test_plan.md`
- 计划样本数：100 条
- 覆盖要求：不少于 5 个法律领域，每个领域 20 条
- 后续数据文件：
  - Day 2：`data/test_samples_part1.json`（30 条）
  - Day 3：`data/test_samples_part2.json`（累计 70 条）
  - Day 4：`data/test_samples_full.json`（完整 100 条）

---

## 2. 测试目标

本测试用于评价法律法规智能问答系统在以下方面的表现：

1. 能否根据用户自然语言问题检索到正确的法律条文；
2. 能否基于检索结果生成正确、完整、可追溯的回答；
3. 能否处理直接法条查询、具体情景判断、适用条件与例外、权利救济及综合分析问题；
4. 能否避免编造法律名称、条文编号、适用条件或法律后果；
5. 能否区分“检索错误”和“生成错误”，为后续系统改进提供依据。

---

## 3. 测试样本字段

每条测试样本严格使用组长规定的 5 个字段：

```json
{
  "question": "用户提出的法律问题",
  "expected_answer": "根据项目法律文本人工编写的标准答案",
  "retrieved_text": "调用 /search 接口后返回的法律原文",
  "model_output": "调用 /rag 接口后返回的系统答案",
  "evaluation_note": "人工对检索和回答结果的评价"
}
```

### 3.1 字段填写顺序

| 字段 | Day 1 | Day 2 | Day 3 以后 |
|---|---|---|---|
| `question` | 设计问题框架 | 填写正式问题 | 必要时微调 |
| `expected_answer` | 暂不填写 | 人工依据法条填写 | 复核 |
| `retrieved_text` | 不填写 | 留空 | 从 `/search` 原样复制 |
| `model_output` | 不填写 | 留空 | 从 `/rag` 原样复制 |
| `evaluation_note` | 设计评价规则 | 可写“待测试” | 对照后填写正式评价 |

## 6. 标准答案编写规则 (Day 2)

每条 `expected_answer` 采用以下结构：

> **明确结论 + 法律名称/条文依据 + 适用条件 + 必要例外或限制**

示例结构：

```text
不可以/可以/应当根据具体条件判断。
根据《××法》第×条，……
本题中……符合/不符合该条件，因此……
如存在……情形，还需要进一步判断……
```

## 7. 接口测试规则  (Day 3)

### 7.1 `/search` 测试

将 `question` 发送给 `/search`，把系统返回的法律原文原样写入：

```json
"retrieved_text": "……"
```


### 7.2 `/rag` 测试

将相同 `question` 发送给 `/rag`，把系统最终返回内容原样写入：

```json
"model_output": "……"
```

即使模型回答错误，也不得修改。

### 7.3 建议先做冒烟测试

接口开放后先选 5 条测试：

- 2 条简单直接查询；
- 2 条中等情景题；
- 1 条困难综合题。

确认接口请求格式、返回字段、中文编码和超时处理均正常后，再批量测试至少 20 条。

## 8. 评价规则

`evaluation_note` 使用统一模板：

```text
检索：命中 / 部分命中 / 未命中。
回答：准确 / 基本准确 / 部分正确 / 错误。
说明：结论是否正确；法条是否正确；条件是否完整；是否存在编造。
错误阶段：无 / 检索阶段 / 生成阶段 / 测试题本身存在歧义。
```

### 8.1 回答等级

| 等级 | 判定标准 |
|---|---|
| 准确 | 结论、依据、关键条件均正确，无明显遗漏 |
| 基本准确 | 主要结论正确，但遗漏次要条件或表达不够完整 |
| 部分正确 | 仅部分结论或法条正确，存在影响使用的遗漏 |
| 错误 | 主要结论错误、引用错误或出现明显法律幻觉 |

### 8.2 错误归因

- **检索错误**：没有召回正确法律或条文；
- **生成错误**：检索正确，但模型理解、归纳或结论错误；
- **完整性错误**：主要结论正确，但遗漏重要条件、例外或责任；
- **引用错误**：法律名称、条文编号或引用内容不一致；
- **幻觉错误**：编造不存在的法律、条文或法律后果；
- **题目问题**：问题事实不足、存在歧义或超出知识库范围。
